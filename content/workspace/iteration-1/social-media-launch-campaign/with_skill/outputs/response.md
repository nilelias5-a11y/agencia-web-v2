## Open Graph Image para la landing de lanzamiento

Cuando compartas el link de tu landing en redes, LinkedIn y Twitter generan automáticamente una preview con imagen. Sin configurar la OG image, usarán el logo genérico o nada. Con `next/og` puedes generar una imagen dinámica y profesional.

```typescript
// app/launch/opengraph-image.tsx
import { ImageResponse } from 'next/og'

export const size = { width: 1200, height: 630 }
export const contentType = 'image/png'

export default function OgImage() {
  return new ImageResponse(
    <div
      style={{
        background: 'linear-gradient(135deg, #0f172a 0%, #1e3a5f 50%, #0e4f6e 100%)',
        width: '100%',
        height: '100%',
        display: 'flex',
        flexDirection: 'column',
        justifyContent: 'space-between',
        padding: '60px 80px',
        fontFamily: 'sans-serif',
      }}
    >
      {/* Header: badge de lanzamiento */}
      <div
        style={{
          display: 'flex',
          alignItems: 'center',
          gap: 12,
        }}
      >
        <span
          style={{
            background: '#3b82f6',
            color: 'white',
            fontSize: 18,
            fontWeight: 700,
            padding: '6px 16px',
            borderRadius: 999,
            letterSpacing: '0.05em',
            textTransform: 'uppercase',
          }}
        >
          🚀 Lanzamiento
        </span>
      </div>

      {/* Cuerpo: nombre del producto y propuesta de valor */}
      <div style={{ display: 'flex', flexDirection: 'column', gap: 20 }}>
        <h1
          style={{
            color: 'white',
            fontSize: 64,
            fontWeight: 800,
            lineHeight: 1.1,
            margin: 0,
          }}
        >
          [Nombre del SaaS]
        </h1>
        <p
          style={{
            color: '#93c5fd',
            fontSize: 28,
            margin: 0,
            fontWeight: 400,
            maxWidth: 800,
          }}
        >
          Gestión de proyectos con visibilidad total.
          Sin cambiar de app. Sin reuniones de estado.
        </p>
      </div>

      {/* Footer: CTA + dominio */}
      <div
        style={{
          display: 'flex',
          justifyContent: 'space-between',
          alignItems: 'center',
        }}
      >
        <span style={{ color: '#60a5fa', fontSize: 22, fontWeight: 600 }}>
          40% descuento · Primeras 100 plazas
        </span>
        <span style={{ color: '#475569', fontSize: 20 }}>
          tudominio.com/launch
        </span>
      </div>
    </div>
  )
}
```

Este archivo va en `app/launch/opengraph-image.tsx`. Next.js lo detecta automáticamente y lo sirve en la meta tag `og:image` de la ruta `/launch`. Sin configuración adicional.

**Para verificar que funciona:**
```
https://www.opengraph.xyz/url/https%3A%2F%2Ftudominio.com%2Flaunch
```

## Nota de adaptación

**Si el engagement en D-7 y D-5 es bajo (menos de 10 likes en LinkedIn, menos de 5 RT en Twitter):** en D-3 abandona el misterio y revela el nombre del producto — el interés tiene que construirse sobre algo concreto, no sobre promesas vagas.

**Si el hilo de lanzamiento (D0) no genera clics al link:** en las primeras 2 horas del lanzamiento añade un reply con una captura de pantalla del producto o un GIF de 10 segundos mostrando la interfaz — el contenido visual multiplica el CTR en Twitter entre 2x y 3x respecto al texto solo.
